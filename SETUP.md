# دليل الإعداد — نظام دار روضة القرآن

نظام إدارة متكامل (Streamlit) فوق قاعدة بيانات Google Sheets، مجاني بالكامل،
بواجهة عربية RTL بهوية الدار. هذا الدليل يشرح التشغيل المحلي ثم النشر المجاني.

---

## 1) ما الذي يفعله النظام

| القسم | الوظيفة |
|------|---------|
| 📊 لوحة المتابعة | مؤشرات الطلاب والمحفظين والمالية + رسوم بيانية |
| 👨‍🎓 الطلاب | قائمة + إضافة طالب (ربط ولي الأمر، كود تلقائي، عمر تلقائي) |
| 👩‍🏫 المحفظون | قائمة + إضافة + كشف أداء بالساعة |
| 📝 التسجيلات | ربط الطالب بالمحفظ + **النمط الأسبوعي** (يشغّل توليد الحصص) |
| 📅 الحصص والإدخال | **توليد حصص الشهر بضغطة**، ترحيل شهري جماعي، تسجيل/تقييم الحصة |
| 💰 المالية | الإيراد، رواتب المحفظين (×1.01 فودافون + تقريب لـ5)، الأرباح، الفواتير |
| ⭐ التقييمات | رضا أولياء الأمور + ملخص تقييم المحفظين + روابط النماذج |
| 📄 التقارير والتقاويم | تقاويم وتقارير PDF احترافية بالهوية للطالب والمحفظ |

**كيف نقتل التكرار الشهري؟** بدل إعادة كتابة أعمدة الحصص كل شهر: تُحدَّد مرة واحدة
في التسجيل أيام الأسبوع + الوقت + المدة، ثم زر واحد يولّد كل حصص الشهر، وزر آخر
يولّدها لكل التسجيلات النشطة دفعة واحدة.

## 2) التشغيل المحلي (تجربة فورية)

```powershell
cd E:\Dar
pip install -r requirements.txt
streamlit run app.py
```

- الدخول الافتراضي: **admin / dar2026** (غيّره في `.streamlit/secrets.toml`).
- **وضع التجربة دون اتصال:** لو وُجد ملف `data/قاعدة_البيانات.xlsx` يقرأ منه النظام
  تلقائيًا (نسخة من ملف Google). مفيد للعرض والتدريب. أو حدّد مسارًا عبر متغيّر
  البيئة `DAR_LOCAL_XLSX`.
- في هذا الوضع تعمل كل الشاشات والتقارير، لكن **الحفظ في Google معطّل** (قراءة فقط)
  حتى تُفعّل إحدى طريقتي الكتابة في الخطوة 3.

## 3) تفعيل الحفظ المباشر في Google Sheets

يدعم النظام طريقتين للكتابة — اختر واحدة. القراءة تعمل دائمًا عبر رابط CSV العام.

### 3-أ) ✅ الطريقة المجانية المُوصى بها — Google Apps Script (بدون بطاقة، بدون Google Cloud)

1. افتح ورقتك على Google Sheets → **Extensions ▸ Apps Script**.
2. احذف الكود الموجود والصق محتوى الملف `apps_script/Code.gs` (من المستودع).
3. في السطر الأول غيّر `TOKEN` إلى كلمة سر من اختيارك، ثم **احفظ**.
4. **Deploy ▸ New deployment ▸ ⚙️ ▸ Web app**:
   - *Execute as:* **Me**  |  *Who has access:* **Anyone** → **Deploy**.
   - عند ظهور "Google hasn't verified" (طبيعي لكودك): **Advanced ▸ Go to … ▸ Allow**.
5. انسخ **Web app URL** (ينتهي بـ `/exec`).
6. ضع الرابط ونفس الرمز في إعدادات Streamlit (Secrets) أو `.streamlit/secrets.toml`:
   ```toml
   [apps_script]
   url   = "https://script.google.com/macros/s/XXXX/exec"
   token = "نفس الرمز من Code.gs"
   ```
7. شغّل/أعد تشغيل التطبيق — يظهر "✅ Google (حفظ مباشر)" في الشريط الجانبي.

### 3-ب) بديل — حساب خدمة Google (Service Account)

> ملاحظة: هذه الطريقة قد تطلب تفعيل فوترة/بطاقة في بعض الحسابات؛ استخدم 3-أ لتجنّبها.

1. <https://console.cloud.google.com> → مشروع جديد → فعّل **Google Sheets API** و **Drive API**.
2. **Credentials → Service Account → Keys → Add Key → JSON** → نزّل المفتاح.
3. املأ `[gcp_service_account]` في `secrets.toml` من ملف JSON.
4. شارك الورقة مع `client_email` كـ **محرّر (Editor)**.

> أثناء التطوير المحلي احذف/أعد تسمية `data/قاعدة_البيانات.xlsx` كي يكتب التطبيق
> في Google مباشرة بدل الملف المحلي.

## 4) نماذج التقييم (المعلمون وأولياء الأمور)

الطريقة الأسهل لغير التقنيين: **Google Forms** على الموبايل بدون تسجيل دخول.

1. أنشئ نموذجين في Google Forms:
   - **رضا ولي الأمر:** اسم/كود الطالب، الشهر، تقييم من 1–10، مستوى الرضا، ملاحظات.
   - **تقييم المحفظ بعد الحصة:** الطالب، التاريخ، تقييم الأداء، السورة من–إلى، ملاحظات.
2. في كل نموذج: **Responses → Link to Sheets** ووجّهها لنفس ملف Google (أو ملف
   ردود تقرأه اللوحة عبر أوراق `تقييمات أولياء الأمور` / `ردود تقييم المحفظين`).
3. ضع رابطي النموذجين في `[forms]` بـ `secrets.toml` ليظهرا في شاشة «التقييمات».
4. شارك الروابط عبر واتساب. تنعكس النتائج تلقائيًا في الملخصات والتقارير.

> بديل سريع: يستطيع السكرتير إدخال رضا ولي الأمر وتقييم الحصة مباشرة من اللوحة.

---

## 5) النشر المجاني على Streamlit Community Cloud

1. ارفع المشروع إلى مستودع GitHub (تأكد أن `secrets.toml` و `data/` **غير** مرفوعين).
2. ادخل <https://share.streamlit.io> بحساب GitHub → **New app** → اختر المستودع
   و `app.py`.
3. **Advanced settings → Secrets:** الصق محتوى `secrets.toml` كاملًا.
4. **Deploy** — ستحصل على رابط دائم بكلمة مرور، يعمل من أي جهاز.

---

## 6) ترحيل البيانات من الملف القديم (عند الحاجة)

الملف الجديد منظّم ومُعبّأ بالفعل. لإعادة ترحيل/مراجعة من الملف القديم:

```powershell
python migrate.py --old "مسار الملف القديم.xlsx" --out "data/migrated_review.xlsx"
```

يُنتج ملف مراجعة بالمخطط الجديد (لا يلمس الملف الحيّ) — راجِعه ثم استورده يدويًا.

---

## 7) ملاحظة مهمة عن بيانات الحصص الحالية

حصص الملف الحالي تنقصها قيم **«مدة الحصة (دقيقة)»**، لذا تظهر الساعات والرواتب
والإيراد = صفر حتى تُملأ. الحلول:
- استخدم **توليد الحصص** من اللوحة (يكتب المدة دائمًا)، أو
- املأ عمود «مدة الحصة (دقيقة)» للحصص الموجودة، أو
- استخدم بيانات الترحيل (التي تتضمن المدد) كأساس.

المالية تُحسب فقط للحصص التي حالتها **«تمت»** ولها مدة.
